# degen0x ramp-to-8 content rubric

Every content page on degen0x is scored against an 8-criterion rubric. The
target is a mean score ≥ 8.0 across all `/learn`, `/compare`, and `/reviews`
pages.

Scores are 1-10, with 7.5 as the hard floor (pre-commit gate blocks merges
below this threshold).

## The 8 criteria

| # | Criterion      | What raises the score                                          |
|---|----------------|---------------------------------------------------------------|
| 1 | E-E-A-T         | `AuthorAttribution` byline · Article schema · `AuthoritySources` |
| 2 | Depth           | Word count above type floor (learn ≥ 1500, compare ≥ 1000, reviews ≥ 1200) |
| 3 | Intent match    | `<h1>` present · ≥ 3 `<h2>` sections · primary keyword in title |
| 4 | Originality     | No programmatic title prefixes (e.g., "Everything You Need to Know") |
| 5 | Readability     | `AutoTOC` · `ReadingTime` · short paragraphs and scannable structure |
| 6 | Freshness       | `LastUpdated` stamp · `dateModified` in schema                 |
| 7 | Multimedia      | `<Diagram>` SVG · `<LiveMiniStat>` · relevant imagery          |
| 8 | Internal linking| `RelatedContent` cluster · ≥ 3 contextual `href="/"` links      |

The overall page score is the mean of the 8 criteria.

## Scoring a single page

```bash
python3 scripts/score_content.py src/app/learn/what-is-bitcoin/page.tsx
```

## Scoring every content page

```bash
python3 scripts/score_content.py --all
```

## Scoring only staged pages (pre-commit)

```bash
python3 scripts/score_content.py --staged --floor 7.5
```

## Installing the pre-commit hook

```bash
ln -s ../../scripts/precommit-content-gate.sh .git/hooks/pre-commit
chmod +x scripts/precommit-content-gate.sh
```

Bypass once (emergency hotfix, etc.):

```bash
CONTENT_SKIP=1 git commit -m "emergency fix"
```

## Weekly review cadence

Once per week, run:

```bash
python3 scripts/score_content.py --all --json > reports/content-score-$(date +%F).json
```

…and archive the diff vs the previous week under `reports/`. Any page whose
score drops by more than 1.0 is a regression flag.

## Integrating with `degen-page-scaffold`

New pages generated by the scaffold skill MUST include:

- `AuthorAttribution` (or handoff to the content writer)
- `AutoTOC` + `ReadingTime`
- `LastUpdated pathKey="..."`
- At least one `Diagram` slug OR `LiveMiniStat` OR inline SVG
- `RelatedContent`
- A topical `AuthoritySources` block

The scaffold template enforces this via the scoring gate — a scaffold that
produces a page below floor 7.5 fails the pre-commit hook.
